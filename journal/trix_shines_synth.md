## The Architecture of Trust

Three pillars. Each load-bearing. None optional.

```
                    ┌─────────────────────┐
                    │   TRUSTWORTHY AI    │
                    └─────────────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          │                   │                   │
          ▼                   ▼                   ▼
   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
   │   FROZEN    │    │   TERNARY   │    │   LEARNED   │
   │   SHAPES    │    │   WEIGHTS   │    │   ROUTING   │
   └─────────────┘    └─────────────┘    └─────────────┘
          │                   │                   │
          ▼                   ▼                   ▼
    Provability        Reproducibility      Explainability
```

**Frozen Shapes** → The "what" of computation is fixed. Mathematical. Verifiable.

**Ternary Weights** → The "how" of hardware is universal. {-1, 0, +1}. Same everywhere.

**Learned Routing** → The "which" of selection is flexible. Trainable. But constrained.

Remove any pillar and trust collapses.

## The Honest Gaps

The vision is clear. The foundation is solid. Gaps remain:

1. **Training at Scale**
   - Routing-only learning is theorized (Gradient Truth)
   - Not yet demonstrated on meaningful tasks
   - This is R&D work, not implementation

2. **Expressiveness Mapping**
   - We know TriX is Turing-complete for deterministic functions
   - We don't know exactly where the edges are for "neural" tasks
   - Empirical exploration needed

3. **Tooling**
   - No training framework
   - No model zoo
   - No benchmarks on trust-relevant tasks
   - Developer experience doesn't exist yet

These aren't blockers. They're the roadmap.
