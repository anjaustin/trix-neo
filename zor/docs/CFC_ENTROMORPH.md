## Overview

### What is CfC?

**Closed-form Continuous-time (CfC)** networks are a class of neural networks developed by Liquid.ai (MIT CSAIL) that solve ordinary differential equations (ODEs) in closed form rather than through numerical integration.

Traditional recurrent networks (LSTMs, GRUs) process sequences step-by-step. Liquid Neural Networks (LNNs) model continuous-time dynamics but require ODE solvers. CfC networks provide the best of both worlds: **continuous-time dynamics with closed-form solutions**.

### What is TriX?

**TriX** is a computational architecture for **deterministic neural computation** using frozen mathematical operations. Instead of learned weights in a black box, TriX uses:

- **Frozen Shapes**: Mathematical operations that never change (XOR = `a + b - 2ab`)
- **Learned Routing**: Signatures determine which shapes fire
- **Verifiable Output**: Same input always produces same output

### The Convergence

By combining CfC dynamics with TriX topology, we create networks that are:

| Property | CfC | TriX | Combined |
|----------|-----|------|----------|
| Compact | 19 neurons drive a car | Fits in L1 cache | **228 bytes** |
| Fast | No ODE solver | No interpretation | **206 ns/step** |
| Interpretable | Continuous dynamics | Frozen math | **Causal traceability** |
| Evolvable | Differentiable | Mutable | **65M mutations/sec** |

## Components

### 1. cfc_shapes.h — The CfC Tile

The core CfC cell implementation as frozen TriX shapes.

**Key Types:**

```c
CfCParams           // Cell parameters (weights, tau)
CfCParamsFixed      // Cell with precomputed decay
CfCOutputParams     // Output projection parameters
CfCNetwork          // Complete network (cell + output)
```

**Key Functions:**

```c
trix_cfc_cell()           // Single cell forward pass
trix_cfc_cell_fixed()     // Fixed time step variant
trix_cfc_forward()        // Process sequence
trix_cfc_output()         // Project to output
trix_cfc_output_softmax() // Project to probabilities
trix_cfc_infer()          // Full inference pipeline
```

**The CfC Equation:**

```
h(t) = (1 - gate) * h_prev * decay + gate * candidate

Where:
  gate      = σ(W_gate @ [x; h_prev] + b_gate)
  candidate = tanh(W_cand @ [x; h_prev] + b_cand)
  decay     = exp(-Δt / τ)
```

### 2. entromorph.h — The Evolution Engine

Genetic algorithm for CfC networks in pure C.

**Key Types:**

```c
LiquidGenome        // Mutable genome with all weights
EntroRNG            // Fast PRNG (Xorshift64)
MutationParams      // Mutation rate configuration
EvolutionParams     // GA configuration
```

**Key Functions:**

```c
entro_genesis()              // Create random genome
entro_mutate()               // Mutate genome in-place
entro_crossover_uniform()    // Breed two genomes
entro_crossover_blend()      // Interpolate genomes
entro_evaluate_mse()         // Fitness: mean squared error
entro_evaluate_classification() // Fitness: accuracy
entro_select_tournament()    // Tournament selection
entro_find_elite()           // Find top K genomes
entro_evolve_generation()    // One generation step
entro_export_header()        // Freeze winner to .h file
```

### 3. shapefabric.h — The Binary Fabric

Zero-copy executable graphs for maximum evolution speed.

**Key Types:**

```c

ShapeNode           // 16-byte computation node
ShapeGraph          // Executable graph
ShapeGraphHeader    // Binary serialization header
```

**Key Functions:**

```c

```

### 4. lnn2trix.py — The Compiler

Converts trained PyTorch/NumPy weights to frozen C headers.

**Usage:**

```bash
# From PyTorch checkpoint
python lnn2trix.py model.pt --name=my_chip

# From NumPy weights
python lnn2trix.py weights.npz --name=driver

# With fixed time step (precomputes decay)
python lnn2trix.py model.pt --name=sensor --dt=0.01 --fixed

# Generate example for testing
python lnn2trix.py --example --name=test --input-dim=4 --hidden-dim=8 --output-dim=3
```

**Outputs:**

- `{name}_weights.h` — Frozen weight arrays
- `{name}_chip.h` — Ready-to-use chip API

```c
#include "liquid_demo_chip.h"

    printf("Class 0: %.2f%%\n", probs[0] * 100);
    printf("Class 1: %.2f%%\n", probs[1] * 100);
    printf("Class 2: %.2f%%\n", probs[2] * 100);

    return 0;

```

### 2. Evolve a New Network

```c
#include "trixc/entromorph.h"

#define POP_SIZE 64

int main() {
    // Populations (double buffer)
    LiquidGenome pop_a[POP_SIZE], pop_b[POP_SIZE];
    LiquidGenome *current = pop_a, *next = pop_b;

    // RNG
    EntroRNG rng;
    entro_rng_seed(&rng, 42);
    uint32_t next_id = POP_SIZE;

    // Initialize population
    for (int i = 0; i < POP_SIZE; i++) {
        entro_genesis(&current[i], 4, 8, 3, &rng, i);
    }

    // Evolution parameters
    EvolutionParams params = EVOLUTION_DEFAULT;

    // Evolve
    for (int gen = 0; gen < 1000; gen++) {
        // Evaluate fitness (your fitness function here)
        for (int i = 0; i < POP_SIZE; i++) {
            current[i].fitness = evaluate_fitness(&current[i]);
        }

        // Evolve one generation
        entro_evolve_generation(current, next, POP_SIZE,
                                &params, &rng, &next_id, gen);

        // Swap buffers
        LiquidGenome *tmp = current;
        current = next;
        next = tmp;
    }

    // Find and export winner
    int best = 0;
    for (int i = 1; i < POP_SIZE; i++) {
        if (current[i].fitness > current[best].fitness) best = i;
    }

    FILE* f = fopen("winner_evolved.h", "w");
    entro_export_header(&current[best], "WINNER", f);
    fclose(f);

    return 0;
}
```

### 3. Zero-Copy Evolution with ShapeGraph

```c
#include "trixc/shapefabric.h"

int main() {
    // Allocate graph
    ShapeNode nodes[256];
    float regs[128];
    ShapeGraph graph;

    shape_graph_init(&graph, nodes, regs, 256, 128);

    // Build CfC network
    shape_graph_set_inputs(&graph, 4);

    uint32_t states[8];
    for (int i = 0; i < 8; i++) {
        states[i] = shape_alloc_reg(&graph);
    }

    float taus[8] = {1.0f, 1.0f, 1.0f, 1.0f, 1.0f, 1.0f, 1.0f, 1.0f};
    uint32_t inputs[8] = {0, 1, 2, 3, 0, 1, 2, 3};
    shape_build_cfc_layer(&graph, inputs, states, 8, taus);

    shape_graph_set_outputs(&graph, 2);

    // Execute
    graph.registers[0] = 0.5f;  // Input 0
    graph.registers[1] = -0.3f; // Input 1
    graph.registers[2] = 0.1f;  // Input 2
    graph.registers[3] = 0.8f;  // Input 3

    shape_execute_graph(&graph, 0.1f);  // dt = 0.1

    printf("Output 0: %f\n", graph.registers[graph.output_start]);
    printf("Output 1: %f\n", graph.registers[graph.output_start + 1]);

    // Mutate (zero-copy!)
    EntroRNG rng;
    entro_rng_seed(&rng, 12345);

    for (int i = 0; i < 1000000; i++) {
        uint32_t node = entro_rng_int(&rng, graph.node_count);
        uint32_t new_input = entro_rng_int(&rng, graph.reg_count);
        shape_mutate_rewire(&graph, node, new_input);
    }

    return 0;
}
```

## Theory

### The CfC Equation

The standard CfC cell computes:

```
h(t) = (1 - g) · h(t-1) · exp(-Δt/τ) + g · c

Where:
  g = σ(W_g · [x; h] + b_g)     (gate)
  c = tanh(W_c · [x; h] + b_c)  (candidate)
  τ = time constant
  Δt = time step
```

This is a **closed-form solution** to the ODE:

```
dh/dt = (1/τ) · (σ(f(x,h)) · g(x,h) - h)
```

### Why Closed-Form Matters

**Standard ODE solvers** (Euler, RK4) require multiple evaluations per time step:
- Euler: 1 evaluation, O(Δt) error
- RK4: 4 evaluations, O(Δt⁴) error

**CfC closed-form** jumps directly to the answer:
- 1 evaluation, exact solution
- Works for any Δt without accuracy loss

### The TriX Connection

TriX decomposes all computation into **5 Primes**:

| Prime | Operation | CfC Usage |
|-------|-----------|-----------|
| ADD | a + b | State mixing |
| MUL | a × b | Weight application, gating |
| EXP | eᵃ | Sigmoid, decay |
| MAX | max(a,b) | ReLU (MAX with 0) |
| CONST | c | Weights, biases, tau |

The CfC cell is a **composition of shapes**:

```
GEMM ──► Sigmoid ──► Gate ────────┐
                                  │
GEMM ──► Tanh ──► Candidate ──┬───┼──► ADD ──► h_new
                              │   │
h_prev ──► EXP(-dt/τ) ──► Decay ──┘
```

### Determinism Guarantee

Because all shapes are **frozen polynomials** (or transcendentals like exp/tanh), the same input always produces the same output:

1. No random operations
2. No floating-point non-determinism (careful implementation)
3. No data-dependent branching
4. No external state

## Genesis: The Foundry

The Foundry (`zor/foundry/`) contains **Genesis**, a seed generator that evolves CfC networks from scratch.

### Key Discoveries

#### 1. The DC Trap

Random CfC networks often converge to outputting a constant value. This is a local optimum where gradient-based methods would get stuck, but evolution can escape with a **variance penalty**:

```c
// Kill flat-line outputs
if (output_variance < 0.01f) {
    return 0.0f;  // Zero fitness
}
```

#### 2. The Phase-Lock Insight

CfC neurons are **smoothers**, not predictors. They naturally produce phase-shifted versions of their input. Instead of fighting this, we reward correlation at any phase:

```c
// Cross-correlation fitness
float best_corr = 0;
for (int lag = -5; lag <= 5; lag++) {
    float corr = correlation(output, input, lag);
    if (corr > best_corr) best_corr = corr;
}
return best_corr;  // Reward best phase-shifted correlation
```

Result: Networks converge in 5 generations with r > 0.97.

#### 3. Dual-Tau Detection

Single-tau trackers miss slow drift. The solution: combine a fast CfC tracker (τ ~ 1s) with a slow EMA anchor (τ ~ 30s).

See [ZIT_DETECTION.md](ZIT_DETECTION.md) for details.

### Usage

```bash
cd zor
make genesis-sine      # Evolve a phase-locked tracker
make genesis-anomaly   # Evolve an anomaly detector
```

Generated seeds are self-contained `.h` files that can be deployed anywhere with zero dependencies.
