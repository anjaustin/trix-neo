# The Forge — EntroMorphic Chip Compiler

> **"Data in, chip out."**

The Forge is the product layer of the EntroMorphic Stack. It takes a CSV file containing time-series data and evolves a frozen chip that can track and detect anomalies in that signal.

**No ML expertise required. No Python. No GPUs. Just data.**

---

## Quick Start

```bash
# Build the forge
make forge

# Train a chip on your data
./build/forge train -i sensor_data.csv -o my_chip.h

# Use in your firmware
#include "my_chip.h"
float state[MY_CHIP_REGS] = {0};
float output = MY_CHIP_step(raw_sensor_reading, state);
```

## How It Works

```
┌─────────────────────────────────────────────────────────────────┐
│                         THE FORGE                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   CSV Data                                                       │
│   ┌──────────┐                                                   │
│   │ 45.2     │                                                   │
│   │ 47.1     │    ┌──────────────┐    ┌──────────────────┐      │
│   │ 43.8     │───►│     AGC      │───►│   EntroMorph     │      │
│   │ 46.5     │    │ Normalization│    │   Evolution      │      │
│   │ ...      │    └──────────────┘    └────────┬─────────┘      │
│   └──────────┘                                  │                │
│                                                 ▼                │
│                                        ┌──────────────────┐      │
│                                        │   Frozen Chip    │      │
│                                        │   (C Header)     │      │
│                                        │                  │      │
│                                        │  - 98 nodes      │      │
│                                        │  - 99 registers  │      │
│                                        │  - ~2KB total    │      │
│                                        │  - AGC baked in  │      │
│                                        └──────────────────┘      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### The Process

1. **Load CSV**: Single column of sensor readings (any scale)
2. **AGC Normalization**: Compute mean/std, normalize to [-1, 1]
3. **Evolution**: Evolve a CfC neural network to track the signal
4. **Export**: Generate a self-contained C header with AGC baked in

## Usage

```bash
./forge train --input <file.csv> [options]
```

### Options

| Option | Short | Default | Description |
|--------|-------|---------|-------------|
| `--input` | `-i` | (required) | Input CSV file |
| `--out` | `-o` | `chip.h` | Output header file |
| `--name` | `-n` | (from filename) | Chip name (uppercase) |
| `--epochs` | `-e` | 5000 | Max generations |
| `--time` | `-t` | 60 | Target time (seconds) |
| `--pop` | `-p` | 256 | Population size |

### Examples

```bash
# Basic usage
./forge train -i vibration.csv -o vibration_chip.h

# Longer training for better accuracy
./forge train -i vibration.csv -o vibration_chip.h -t 120 -e 10000

# Custom chip name
./forge train -i data.csv -o chip.h -n MOTOR_SENSOR
```

## CSV Format

The Forge accepts simple CSV files:

### Single Column (Recommended)
```csv
45.2
47.1
43.8
46.5
48.0
```

### Two Columns (Timestamp + Value)
```csv
timestamp,value
0.0,45.2
0.1,47.1
0.2,43.8
```

The timestamp column is ignored. Only the value column is used.

### Requirements
- Minimum 100 samples
- Maximum 65,536 samples
- Header row is auto-detected and skipped

## Output Format

The Forge generates a self-contained C header file:

```c
/*
 * MOTOR_CHIP — EntroMorphic Frozen Chip
 *
 * Generated by The Forge
 * Fitness: 10649.00
 * Training samples: 4096
 *
 * AGC Normalization (baked in):
 *   Mean: 345.953735
 *   Std:  247.263458
 */

#ifndef MOTOR_CHIP_H
#define MOTOR_CHIP_H

/* Chip Metadata */
#define MOTOR_CHIP_NODES 98
#define MOTOR_CHIP_REGS 99
#define MOTOR_CHIP_FITNESS 10649.00f

/* AGC Constants (baked in) */
#define MOTOR_CHIP_AGC_MEAN 345.9537f
#define MOTOR_CHIP_AGC_STD  247.2635f

/* Frozen neural network data... */

/* Public API */
static inline float MOTOR_CHIP_step(float raw_input, float* state);
static inline void MOTOR_CHIP_reset(float* state);

#endif
```

### Key Features

1. **AGC Baked In**: Pass raw sensor values directly
2. **Zero Dependencies**: Only needs `<math.h>`, `<string.h>`, `<stdint.h>`
3. **Header-Only**: No separate compilation needed
4. **Self-Documenting**: Includes training metadata

## Integration

### Basic Usage

```c
#include "motor_chip.h"

// Declare state (persistent across calls)
float state[MOTOR_CHIP_REGS] = {0};

void loop() {
    float raw = read_sensor();  // e.g., 345.5 mV

    // Execute chip (AGC is automatic)
    float prediction = MOTOR_CHIP_step(raw, state);

    // Compute anomaly score
    float normalized = (raw - MOTOR_CHIP_AGC_MEAN) / MOTOR_CHIP_AGC_STD;
    float delta = fabsf(normalized - prediction);

    // Alert if delta exceeds threshold
    if (delta > 0.112) {  // Second Star Constant
        trigger_alert();
    }
}
```

### Anomaly Detection Pattern

```c
#define THRESHOLD 1.5f  // Adjust based on your data

float detect_anomaly(float raw_value, float* state) {
    float prediction = MOTOR_CHIP_step(raw_value, state);
    float normalized = (raw_value - MOTOR_CHIP_AGC_MEAN) / MOTOR_CHIP_AGC_STD;
    return fabsf(normalized - prediction);
}

// Usage
float anomaly_score = detect_anomaly(sensor_reading, state);
if (anomaly_score > THRESHOLD) {
    // Pattern has changed!
}
```

### Reset State

```c
// Call when you want to forget history
MOTOR_CHIP_reset(state);
```

## Troubleshooting

### "Need at least 100 samples"

Your CSV file is too small. The Forge needs enough data to learn patterns.

### High Zit rate on training data

The threshold may be too tight for your signal. Options:
- Increase training time (`-t 120`)
- Use a higher threshold (mean + 2*std of delta)
- Ensure your data has consistent patterns

### Chip not tracking well

- Increase population size (`-p 512`)
- Increase training time (`-t 120`)
- Check that your data has learnable patterns (not pure noise)

## Architecture

The Forge uses:

1. **EntroMorph Evolution**: Genetic algorithm for topology discovery
2. **CfC Neurons**: Closed-form Continuous-time cells (Liquid Time-Constants)
3. **Gluttony Penalty**: Metabolic regularization prevents state explosion
4. **Anti-Identity Check**: Prevents trivial pass-through solutions
